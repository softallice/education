# 01 · 학사 일정

> 이 강의가 **언제, 몇 주간, 하루에 어떻게** 진행되는지 정리한다. P-학기제의 4교시 편성을 어떻게 운영할지(옵션 A/B)와 12주 캘린더가 핵심이다.

## P-학기제란

가천대학교 3학년은 **P-학기제**를 운영한다. 기존 15주 수업을 재구성하여:

- **12주** — 정규 강의 진행
- **4주** — 몰입형 프로젝트 (한 과목/주제에 집중)

따라서 본 강의도 **개강일(2026-09-01 화)부터 12주간** 강의를 담당하고, 이후 4주는 [몰입형 프로젝트](./03-평가와-몰입형-프로젝트.md)로 운영한다. 15주 → 12주 압축이므로 주당 수업량이 크다(**4교시 편성**). 그래서 하루를 어떻게 운영하느냐가 학습 성패를 가른다.

## 12주 + 4주 배치 (개략)

| 기간 | 블록 | 내용 |
|---|---|---|
| 1–5주 | 블록 1 | 클라우드 네이티브 기반 (로컬 웹앱 → Docker → Compose → Kubernetes → 운영형 배포) |
| 6–8주 | 블록 2 | AI 서비스 개발 (LLM API → 멀티모달/HF → RAG) |
| 9–11주 | 블록 3 | AI Agent (단일 Agent → 도구·MCP 연동 → Multi-Agent) |
| 12주 | 통합 | K8s 통합 배포 + 관측성 + 몰입형 프로젝트 킥오프 |
| 몰입 4주 | — | 팀별 Agentic AI 서비스 구축·배포·발표 |

## 시간표

| 구분 | 요일 | 시간 | 비고 |
|---|---|---|---|
| **임시 배정** | 월요일 1–4교시 | 09:00–12:50 (3시간 50분) | 현재 배정안 |
| **대안 가능** | 화요일 전시간 | 09:00–17:50 | 조정 여지 |

- 주당 **4교시**로 편성되며, 운영 방식은 유연하다: **4시간 연강**(옵션 A) 또는 **2시간씩 분강(2+2)**(옵션 B) 모두 가능.
- 화요일 종일 슬롯을 쓰면 오전/오후 분리 편성(옵션 B)이 자연스럽다.

## 4교시 운영 옵션

주당 4교시(약 3시간 50분)는 집중력 유지가 관건이다. 두 가지 운영안을 제시한다.

### 옵션 A — 연강 (권장, 실습 몰입형)

```text
09:00–10:20  1부  개념·데모      (짧은 이론 + 라이브 코딩/데모)
10:20–10:30  휴식
10:30–11:50  2부  가이드 실습     (강사와 함께 따라 만들기)
11:50–12:00  휴식
12:00–12:50  3부  자기주도 실습·과제 착수 (질의응답 상시)
```

- **장점**: 한 주제를 끊김 없이 "개념 → 구현 → 배포"까지 완결. 실습 중심·누적형 프로젝트 구조와 가장 잘 맞는다.
- 각 주차 문서의 "실습: 단계별 따라하기"가 이 1부/2부/3부 흐름에 대응한다.

### 옵션 B — 분강 (2+2)

- 예: 오전 2교시(개념·데모) + 오후 2교시(실습), 또는 주중 이틀로 분리.
- **장점**: 인지 부하 분산, 실습 과제 사이 소화 시간 확보. 화요일 종일 슬롯을 활용하면 오전/오후 분리 편성이 편하다.

> **운영 권장**: 초반(1–8주) 신규 개념 밀도가 높은 구간은 **옵션 A 연강**으로 몰입하고, 후반(9–12주) Agent·통합 구간은 팀 실습 비중이 커지므로 **분강 또는 연강 내 실습 시간 확대**를 고려한다.

## 몰입형 4주 개요

12주 정규 강의가 끝나면, 남은 **4주는 몰입형 프로젝트**로 운영한다. 12주차에 킥오프한 주제를 팀 단위로 완성한다.

- **주제**: 12주간 키운 `docpilot`을 기반으로, 팀별 **Agentic AI 서비스**를 기획·구축·배포한다.
- **흐름(개략)**: 기획 확정 → 구현(AI 기능 + Agent + 클라우드 배포) → 관측성·운영 점검 → 최종 발표·시연.
- **평가 연계**: 몰입형 결과물은 성적에 크게 반영된다. 세부 배점·산출물·발표 기준은 [03-평가와-몰입형-프로젝트](./03-평가와-몰입형-프로젝트.md)에서 확정한다.
