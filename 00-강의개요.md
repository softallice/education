## 교과 목표 — 3대 축

AI 및 클라우드 환경에서 동작하는 **지능형 응용 서비스**를 직접 만들어 보는 실습 중심 교과목이다. 세 축을 따로 배우지 않고, 하나의 러닝 프로젝트 `docpilot`에 통합해 다룬다.

1. **① 클라우드 네이티브 개발·배포** — Docker와 Kubernetes로 애플리케이션을 컨테이너화하고, 로컬 클러스터(kind/minikube)에 배포·확장·운영한다. 12-Factor App, 마이크로서비스, 무중단 배포(롤아웃/롤백)를 손으로 익힌다.
2. **② AI API 연계 서비스 구현** — OpenAI, Google Gemini, Hugging Face 등 상용/오픈 AI API를 호출해 실제 동작하는 기능(챗, 멀티모달, 임베딩, RAG)을 구현한다. 토큰·비용·스트리밍 같은 실무 관심사도 함께 다룬다.
3. **③ AI Agent · Multi-Agent 프로그래밍 (Agentic AI)** — LLM에 추론+행동 루프(ReAct)를 붙여 Tool/Function calling으로 외부 도구를 쓰게 하고, **MCP(Model Context Protocol)** 로 도구·데이터를 표준 연결하며, 여러 에이전트가 협업하는 Multi-Agent 시스템으로 확장한다.

이 세 축을 통해 **AI 서비스 개발 · 클라우드 배포/운영 · AI Agent 프로그래밍**에 필요한 실무형 역량을 배양한다.

## 수강 후 역량 (학습 성과)

수강을 마치면 다음을 스스로 할 수 있어야 한다.

- [ ] 애플리케이션을 컨테이너로 패키징하고, Docker Compose로 멀티 컨테이너(웹+DB)를 오케스트레이션할 수 있다.
- [ ] 쿠버네티스에 애플리케이션을 배포·노출·확장하고, ConfigMap/Secret·probe·HPA·Ingress로 운영형 배포를 구성할 수 있다.
- [ ] LLM·멀티모달 AI API를 호출해 스트리밍 응답까지 갖춘 실제 서비스를 만들 수 있다.
- [ ] 임베딩과 벡터DB(Chroma/pgvector)를 이용한 검색 증강 생성(RAG)으로 외부 지식을 결합한 응답을 구현할 수 있다.
- [ ] Tool/Function calling과 MCP로 외부 도구·데이터를 연동한 단일 AI Agent를 설계할 수 있다.
- [ ] researcher+writer 같은 역할 분담으로 협업하는 Multi-Agent 워크플로우를 오케스트레이션할 수 있다.
- [ ] 위 결과물을 클라우드 네이티브 방식으로 통합·배포·운영하고, 로그·모니터링·키 관리 관점에서 점검할 수 있다.

## 설계 철학

이 강의의 모든 문서와 실습은 아래 네 가지 관통축을 따른다.

- **실습 중심** — 매주 이론은 최소화하고, 손으로 만든 결과물을 우선한다. 4교시(약 3시간 50분)는 "짧은 개념·데모 → 가이드 실습(따라 만들기) → 자기주도 실습·과제 착수"의 흐름으로 운영한다(상세: [01-학사일정](./01-학사일정.md)의 4교시 운영 옵션).
- **누적형 프로젝트** — 초반 실습이 후반 몰입형 프로젝트의 재료가 되도록 설계했다. 1주차 웹앱 → 컨테이너화 → K8s 배포 → AI 기능 → Agent화 → 통합 배포까지, **매주 같은 `docpilot`이 자란다.** 버리는 코드가 없다.
- **최신 스택 우선** — 실무에서 실제로 쓰는 도구(Docker/K8s, 최신 LLM API, Agent 프레임워크, MCP)를 그대로 사용한다. 고정 스택은 [02-실습환경과-기술스택](./02-실습환경과-기술스택.md)을 참조한다.
- **클라우드 네이티브를 관통 축으로** — AI 기능을 만들되, 항상 "이걸 어떻게 컨테이너화하고 배포·운영할 것인가"로 되돌아온다. AI 두뇌를 붙여도 배포 가능한 서비스여야 한다는 원칙을 12주 내내 유지한다.

> **AI 도구 활용 방침**: 실습 전반에서 코딩 어시스턴트 사용을 권장한다(실무 역량의 일부). 단, 생성된 코드가 "왜 동작하는가"를 설명할 수 있어야 한다.

## 선수 지식 (권장)

필수는 아니지만, 아래를 갖추면 실습 속도가 크게 붙는다. 부족한 부분은 1주차 오리엔테이션과 [02-실습환경과-기술스택](./02-실습환경과-기술스택.md)에서 보완한다.

- **프로그래밍 언어 하나 이상** — 실습 기본 언어는 **Python(3.12)** 이다. 다른 언어 경험이 있으면 문법은 빠르게 따라올 수 있다.
- **기초 웹/HTTP·REST API 개념** — 요청/응답, 상태 코드, JSON 정도. FastAPI 실습에서 바로 쓴다.
- **Git 기본 사용** — clone, commit, push. `docpilot`은 매주 커밋을 쌓아 성장한다.
- (있으면 좋음) 리눅스 셸 기본 명령, 컨테이너/클라우드에 대한 막연한 개념.

## 러닝 프로젝트 소개 — `docpilot`

**`docpilot` = 문서 기반 AI 도우미.** 사용자가 문서를 올리면 그 내용으로 질문에 답하고, 나아가 스스로 도구를 사용해 작업을 수행하는 서비스다. 이 하나의 프로젝트가 12주 전 주차를 관통한다.

왜 하나의 프로젝트를 계속 키우는가?

- **누적성** — 매주 새 개념이 이전 결과물 위에 쌓인다. 배포를 배우면 지난주 앱을 배포하고, RAG를 배우면 이미 배포된 앱에 RAG를 붙인다. 개념이 고립되지 않는다.
- **현실성** — 실제 서비스는 컨테이너화·배포·AI 기능·운영이 뒤섞여 있다. `docpilot`은 이 뒤섞임을 그대로 경험하게 한다.
- **몰입형 연결** — 12주차 통합 배포된 `docpilot`은 4주 몰입형 프로젝트의 출발점이 된다.

주차별 진화표:

| 주 | docpilot 진화 | 결과물 |
|---|---|---|
| 1 | FastAPI `GET /`("Hello, DocPilot") + `/health`, 로컬 uvicorn 실행 | 로컬 웹앱 |
| 2 | Dockerfile로 컨테이너화, 이미지 빌드·실행·레지스트리 푸시 | 컨테이너 이미지 |
| 3 | docker-compose: web + PostgreSQL, `/documents`(업로드 메타 DB 저장) | 멀티 컨테이너 |
| 4 | kind/minikube에 Deployment + Service 배포, kubectl 확인 | K8s 배포 |
| 5 | ConfigMap/Secret, probe, HPA, Ingress, 롤아웃/롤백 | 운영형 배포 |
| 6 | `/chat`: OpenAI/Gemini LLM 호출 + 스트리밍 + 프롬프트 | LLM 챗 |
| 7 | 멀티모달(이미지/음성) + Hugging Face 임베딩 모델 도입 | 멀티모달 기능 |
| 8 | RAG: 문서 임베딩 → 벡터DB(Chroma 또는 pgvector) → 검색 → `/ask` | RAG 서비스 |
| 9 | Agent 기초: tool/function calling, ReAct, `/agent`가 도구 호출 | 단일 Agent |
| 10 | 실제 API·DB·파일 도구 연동 + **MCP** 서버로 docpilot 도구 노출 | Agentic 서비스 |
| 11 | Multi-Agent: researcher + writer 협업으로 보고서 생성 | Multi-Agent |
| 12 | 전체를 K8s에 통합 배포 + 관측성 + 몰입형 프로젝트 킥오프 | 통합 배포 |

주차별 실습의 실제 단계는 [주차별-커리큘럼/README.md](./주차별-커리큘럼/README.md)에서 확인한다.
