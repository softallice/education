# 클라우드프로그래밍 — 따라-하기 강의 교재

> 문서를 올리면 그 내용으로 답하고, 나아가 스스로 도구를 써서 일을 처리하는 AI 서비스 **`docpilot`** 을 12주에 걸쳐 직접 만든다. 클라우드 네이티브(Docker/K8s) · AI API 연계 · AI Agent/Multi-Agent를 하나의 프로젝트로 관통한다.

## 이 저장소는 무엇인가

이 저장소는 **가천대학교 클라우드공학과 3학년 전공선택 「클라우드프로그래밍」(3학점)** 의 **따라-하기(follow-along) 강의 교재**다. 슬라이드 요약이 아니라, 명령을 한 줄씩 따라 치면 매주 동작하는 결과물이 손에 남도록 설계했다.

- 모든 실습은 러닝 프로젝트 `docpilot` 하나에 누적된다. 1주차의 로컬 웹앱이 12주차에는 K8s에 통합 배포된 Agentic AI 서비스로 자란다.
- 명령마다 "무엇을/왜" 한 줄 설명과 실행 후 **확인**(기대 출력) 지점을 제시한다.
- 서술은 한국어, 코드·명령·식별자는 영문 그대로.

## 러닝 프로젝트 `docpilot` 한눈에

**`docpilot` = 문서 기반 AI 도우미.** 매주 같은 프로젝트가 자란다.

| 주 | docpilot 진화 | 결과물 |
|---|---|---|
| 1 | FastAPI `GET /`("Hello, DocPilot") + `/health`, 로컬 uvicorn 실행 | 로컬 웹앱 |
| 2 | Dockerfile로 컨테이너화, 이미지 빌드·실행·레지스트리 푸시 | 컨테이너 이미지 |
| 3 | docker-compose: web + PostgreSQL, `/documents`(업로드 메타 DB 저장) | 멀티 컨테이너 |
| 4 | kind/minikube에 Deployment + Service 배포, kubectl 확인 | K8s 배포 |
| 5 | ConfigMap/Secret, probe, HPA, Ingress, 롤아웃/롤백 | 운영형 배포 |
| 6 | `/chat`: OpenAI/Gemini LLM 호출 + 스트리밍 + 프롬프트 | LLM 챗 |
| 7 | 멀티모달(이미지/음성) + Hugging Face 임베딩 모델 도입 | 멀티모달 기능 |
| 8 | RAG: 문서 임베딩 → 벡터DB(Chroma 또는 pgvector) → 검색 → `/ask` | RAG 서비스 |
| 9 | Agent 기초: tool/function calling, ReAct, `/agent`가 도구 호출 | 단일 Agent |
| 10 | 실제 API·DB·파일 도구 연동 + **MCP** 서버로 docpilot 도구 노출 | Agentic 서비스 |
| 11 | Multi-Agent: researcher + writer 협업으로 보고서 생성 | Multi-Agent |
| 12 | 전체를 K8s에 통합 배포 + 관측성 + 몰입형 프로젝트 킥오프 | 통합 배포 |

## 커리큘럼 아크

```text
[블록 1] 1–5주   클라우드 네이티브 기반
  week01 로컬 웹앱 ─▶ week02 컨테이너 ─▶ week03 Compose+DB ─▶ week04 K8s 배포 ─▶ week05 운영형 배포
        │
        ▼  (같은 docpilot에 AI 두뇌를 붙인다)
[블록 2] 6–8주   AI 서비스 개발
  week06 LLM 챗 ─▶ week07 멀티모달/HF ─▶ week08 RAG
        │
        ▼  (응답만 하던 서비스가 스스로 행동한다)
[블록 3] 9–11주  AI Agent / Agentic AI
  week09 단일 Agent ─▶ week10 도구·MCP 연동 ─▶ week11 Multi-Agent
        │
        ▼
[통합]   12주     K8s 통합 배포 + 관측성 + 몰입형 프로젝트 킥오프
        │
        ▼
[몰입]   4주      팀별 Agentic AI 서비스 구축·배포·발표
```

## 시작하기

1. **먼저 실습 환경을 세팅한다.** [02-실습환경과-기술스택](./02-실습환경과-기술스택.md)을 따라 Python 3.12, Docker, kubectl/kind, 필요한 API 계정·키를 준비한다.
2. **1주차부터 시작한다.** [주차별-커리큘럼/README.md](./주차별-커리큘럼/README.md)에서 `week01` 문서를 열어 `docpilot`의 첫 커밋을 만든다.
3. 각 주차 문서는 `학습 목표 → 사전 준비 → 개념 → 단계별 따라하기 → 트러블슈팅 → 과제 → 체크리스트 → 다음 주 예고` 순서다. 위에서 아래로 그대로 따라간다.

> 강의 전체 흐름과 배경은 [00-강의개요](./00-강의개요.md), 일정·시간표는 [01-학사일정](./01-학사일정.md)에서 확인한다.
