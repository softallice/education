개념 ① / LLM API란 무엇인가

모델을 HTTP로 부른다

LLM은 "지금까지의 대화"를 입력받아 "다음에 올 말"을 확률적으로 이어 쓰는 모델이다. 우리는 이 모델을 직접 돌리지 않고 HTTP API로 호출한다.

우리 서버
(docpilot)— HTTPS 요청 (messages + 파라미터) → LLM 제공자
우리 서버← 응답 (생성 텍스트 + usage 토큰) — LLM 제공자
인증

Authorization: Bearer <KEY>(SDK가 대신 처리). 키는 곧 "돈" — 코드·깃에 절대 금지.

요청

모델 이름 + messages 배열(역할별 메시지) + 파라미터(temperature 등).

응답

생성 텍스트 + usage(입력/출력 토큰 수).

개념 ②③ / 토큰·비용 · 스트리밍

비용은 토큰, 체감은 스트리밍

② 토큰과 비용

토큰: 모델이 텍스트를 자르는 단위. 영어 1토큰 ≈ 4글자, 한글은 1글자 ≈ 1~2토큰.

비용 = (입력 토큰 × 입력 단가) + (출력 토큰 × 출력 단가). 소형 모델이 훨씬 저렴.

컨텍스트(입력)를 길게 넣을수록 매 호출 비용↑ → week08 RAG에서 "필요한 조각만" 넣는 이유.

③ 스트리밍(streaming)

기본 호출은 완성된 답 전체를 한 번에 받는다 → 답이 길면 몇 초간 빈 화면.

스트리밍은 토큰을 조각조각 내려준다(ChatGPT 타이핑 효과). 흔히 SSE: text/event-stream으로 data: ... 라인을 계속 흘린다.

data: 클data: 라data: 우드data: …done
개념 ④ / 프롬프트 엔지니어링 기초

messages는 역할을 가진 배열

role의미 · 예
system모델의 정체성·규칙·톤. 대화 맨 앞 1개.
"너는 docpilot, 모르면 모른다고 답한다."
user사용자의 실제 입력. "배포 방법 알려줘"
assistant모델의 이전 답변(히스토리를 이어줄 때 넣음).
파라미터역할 · 감 잡기
temperature무작위성/창의성. 0=결정적.
사실 0~0.3, 브레인스토밍 0.7~1.0
max_tokens출력 길이 상한. 비용·응답시간 제어에 필수
top_p누적확률 샘플링(temperature 대안). 보통 temperature만 조절

핵심 원칙: 지시는 구체적으로, 형식은 명시적으로, 예시를 곁들이면 더 안정적.

2부 · 4단계

app/llm.py — 프로바이더 추상화

app/llm.py — messages 빌더 + OpenAI
from collections.abc import AsyncIterator
from openai import AsyncOpenAI
from app.config import Settings, get_settings

Messages = list[dict[str, str]]
DEFAULT_SYSTEM_PROMPT = (
    "You are docpilot, a helpful document "
    "assistant. Be concise. Answer in Korean.")


def build_messages(user_text, system=None,
                   history=None) -> Messages:
    # system → (history) → user 순서
    msgs = [{"role": "system",
             "content": system or DEFAULT_SYSTEM_PROMPT}]
    if history: msgs.extend(history)
    msgs.append({"role": "user", "content": user_text})
    return msgs


async def _openai_stream(msgs, s: Settings, temp):
    client = AsyncOpenAI(api_key=s.openai_api_key)
    stream = await client.chat.completions.create(
        model=s.openai_model, messages=msgs,
        temperature=temp, stream=True)
    async for chunk in stream:
        delta = chunk.choices[0].delta.content
        if delta: yield delta
얇은 추상화 계층

OpenAI/Gemini/Ollama를 하나의 인터페이스(complete, stream)로 감싼다. /chat은 이 모듈만 알면 되고 프로바이더 세부는 몰라도 된다.

_gemini_*는 google.genai Client로 system_instruction+contents로 변환(assistant→model), _ollama_*는 httpx로 /api/chat 호출 — 같은 시그니처로 구현.

공개 API — 프로바이더 디스패치

app/llm.py — complete / stream
async def complete(messages, temperature=0.3) -> str:
    s = get_settings()
    if s.llm_provider == "openai":
        return await _openai_complete(messages, s, temperature)
    if s.llm_provider == "gemini":
        return await _gemini_complete(messages, s, temperature)
    if s.llm_provider == "ollama":
        return await _ollama_complete(messages, s, temperature)
    raise ValueError(f"Unknown provider: {s.llm_provider}")


async def stream(messages, temperature=0.3):
    s = get_settings()
    if s.llm_provider == "openai":
        gen = _openai_stream(messages, s, temperature)
    elif s.llm_provider == "gemini":
        gen = _gemini_stream(messages, s, temperature)
    elif s.llm_provider == "ollama":
        gen = _ollama_stream(messages, s, temperature)
    else:
        raise ValueError(f"Unknown provider: {s.llm_provider}")
    async for piece in gen:
        yield piece

app/main.py — 기존 app = FastAPI(...) 아래
from pydantic import BaseModel, Field
from app import llm


class ChatRequest(BaseModel):
    message: str = Field(..., min_length=1,
                         max_length=4000)
    system: str | None = None
    temperature: float = Field(0.3, ge=0.0, le=2.0)


class ChatResponse(BaseModel):
    reply: str
    provider: str


@app.post("/chat", response_model=ChatResponse)
async def chat(req: ChatRequest) -> ChatResponse:
    from app.config import get_settings
    messages = llm.build_messages(req.message,
                                  system=req.system)
    reply = await llm.complete(messages,
                              temperature=req.temperature)
    return ChatResponse(reply=reply,
        provider=get_settings().llm_provider)

/chat/stream — SSE로 토큰 흘리기

app/main.py — 추가
from fastapi.responses import StreamingResponse


@app.post("/chat/stream")
async def chat_stream(req: ChatRequest):
    messages = llm.build_messages(req.message,
                                  system=req.system)

    async def event_generator():
        try:
            async for piece in llm.stream(
                    messages, temperature=req.temperature):
                # SSE 프레임: 'data: <내용>\n\n'
                yield f"data: {piece}\n\n"
        except Exception as exc:
            yield f"event: error\ndata: {exc}\n\n"
        finally:
            yield "event: done\ndata: [DONE]\n\n"

    return StreamingResponse(
        event_generator(),
        media_type="text/event-stream",
        headers={"Cache-Control": "no-cache",
                 "X-Accel-Buffering": "no"})
bash — 버퍼링 없이(-N) 확인
curl -N -X POST localhost:8000/chat/stream \

기대 — 조각이 연달아, 끝에 done

완성본을 한 번에 받지 않고 흘러 내려오면 성공. 프로덕션은 조각을 json.dumps로 감싸거나 sse-starlette의 EventSourceResponse 권장(과제).

static/chat.html — fetch 스트림 읽기
async function ask() {
  const out = document.getElementById("out");
  out.textContent = "";
  const res = await fetch("/chat/stream", {
    method: "POST",
    headers: {"Content-Type": "application/json"},
    body: JSON.stringify({message: q.value}),
  });
  const reader = res.body.getReader();
  const decoder = new TextDecoder();
  while (true) {
    const {value, done} = await reader.read();
    if (done) break;
    decoder.decode(value).split("\n").forEach((line) => {
      if (line.startsWith("data: ")) {
        const t = line.slice(6);
        if (t !== "[DONE]") out.textContent += t;
      }
    });
  }
}
