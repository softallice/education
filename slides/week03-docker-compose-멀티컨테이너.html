Week 03 — Docker Compose 멀티 컨테이너

앱 혼자서는 서비스가 안 된다. docpilot(web)과 PostgreSQL을 Compose로 함께 띄우고, 문서 메타데이터를 DB에 저장한다.

01 / 개요

이번 주가 끝나면 할 수 있는 것

  • 1컨테이너 네트워크로 서비스끼리 이름으로 통신하는 원리를 이해한다.
  • 2볼륨으로 데이터를 컨테이너 수명과 분리해 영속화한다.
  • 3환경변수로 설정을 주입하고(12-Factor #3 Config), 시크릿을 코드에 넣지 않는다.

개념 ① / 왜 Compose인가

여러 컨테이너를 한 파일 · 한 명령으로

docker run을 서비스마다 손으로 치면 포트·네트워크·환경변수·순서를 매번 챙겨야 한다. Compose는 이 전부를 하나의 YAML로 선언하고 docker compose up 한 번에 띄운다.

한 파일에 선언하는 것
  • ·services — 띄울 컨테이너(web, db)
  • ·networks — 서비스끼리 통신하는 가상 네트워크
  • ·volumes — 데이터를 영속화하는 저장소
v2 vs v1 — 반드시 v2

docker compose(공백)가 Compose v2다. 옛 docker-compose(하이픈)는 v1이니 가급적 v2를 쓴다.

bash — 버전 확인
docker compose version   # Docker Compose version v2.x
개념 ② / 네트워크 · 볼륨

web은 이름으로 db에 붙는다

web
docpilot (FastAPI)
build: . · 포트 8000
→ db:5432 서비스 이름 = 호스트명
db
postgres:16
내부 포트 5432
🗄 named volume · pgdata → /var/lib/postgresql/data

네트워크: Compose가 프로젝트별 가상 네트워크를 자동 생성 → web은 IP가 아니라 db라는 이름으로 접속한다(컨테이너 IP는 재시작마다 바뀌지만 이름은 안정적 = DNS 역할, 12-Factor "Backing services").  볼륨: 컨테이너는 일회용이라 지우면 데이터도 사라진다 → pgdata 명명 볼륨에 담아 수명과 분리한다.

개념 ③ / 기동 순서 (흔한 함정)

DB가 준비된 뒤 web을 띄운다

순서를 안 맞추면

web이 DB보다 먼저 떠서 접속 에러(connection refused)가 난다. 앱 시작 시 init_db()가 테이블을 만드는데, 이때 DB가 아직 안 떠 있으면 그대로 죽는다.

해법: db에 healthcheck로 "진짜 접속 가능한 상태"를 확인하고, web은 depends_on: condition: service_healthy로 그때까지 기다린다.

기동 순서
# db가 healthy 될 때까지
db  ──▶ pg_isready OK
web ──▶ 그 다음에 시작

depends_on: condition: service_healthy + healthcheck가 핵심이다. 이게 없으면 web이 DB보다 먼저 떠서 접속 에러가 난다.

개념 ④ / Config & Secret

설정은 환경에서, 비밀번호는 코드 밖에서

환경변수 (Config)

접속 주소·포트·DB 이름처럼 환경마다 달라지는 값. 코드가 아니라 환경에서 주입한다. docpilot은 DATABASE_URL 하나로 DB 접속 정보를 받는다. 12-Factor #3 Config.

시크릿 (Secret)

비밀번호·API 키처럼 유출되면 안 되는 값. 절대 코드/이미지에 하드코딩하지 않는다. 이번 주는 .env로 분리하고 .gitignore로 커밋을 막는다.

철칙

비밀번호를 소스에 적지 않는다. .env는 절대 커밋하지 않는다. 이 두 가지는 이번 주부터 습관으로 굳힌다. (운영에서는 Docker/K8s secret이나 Vault를 쓴다 — 5주차 확장.)

1부 · 2단계

db.py — DB 연결 + Document 모델

db.py — docpilot week03
import os
from sqlalchemy import Column, DateTime, Integer, String, create_engine, func
from sqlalchemy.orm import DeclarativeBase, sessionmaker

# 접속 정보는 코드가 아니라 환경변수에서. 로컬 폴백은 개발 편의용.
DATABASE_URL = os.getenv(
    "DATABASE_URL",
    "postgresql+psycopg://docpilot:docpilot@localhost:5432/docpilot",
)

# pool_pre_ping: 죽은 커넥션을 미리 걸러 재연결 (컨테이너 재시작 대비)
engine = create_engine(DATABASE_URL, pool_pre_ping=True)
SessionLocal = sessionmaker(bind=engine, autoflush=False, autocommit=False)

class Base(DeclarativeBase):
    pass

class Document(Base):
    """업로드된 문서의 메타데이터. 실제 파일 내용이 아니라 '정보'만 저장한다."""
    __tablename__ = "documents"
    id = Column(Integer, primary_key=True, index=True)
    filename = Column(String(255), nullable=False)
    content_type = Column(String(100), nullable=False)
    size_bytes = Column(Integer, nullable=False)
    created_at = Column(DateTime(timezone=True), server_default=func.now())

def init_db() -> None:
    """앱 시작 시 테이블이 없으면 생성. (학습용으로 간단히.)"""
    Base.metadata.create_all(bind=engine)

postgresql+psycopg:// 스킴이 psycopg 3 드라이버를 지정한다. 사용자:비밀번호@호스트:포트/DB이름 순서다.

1부 · 3단계 · 절대 누락 금지

Dockerfile — week02 파일에서 아래 한 줄 추가

1부 · 4단계

main.py 확장 — /documents 저장·조회

main.py — docpilot week03 (핵심 발췌)
from fastapi import Depends, FastAPI, File, HTTPException, UploadFile
from sqlalchemy import select
from sqlalchemy.orm import Session
from db import Document, SessionLocal, init_db

app = FastAPI(title="docpilot", version="0.3.0")

@app.on_event("startup")
def on_startup() -> None:
    init_db()   # healthcheck+depends_on으로 db 준비 후 실행됨

def get_db():
    """요청마다 세션을 열고 끝나면 닫는다(의존성 주입)."""
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()

@app.post("/documents", response_model=DocumentOut, status_code=201)
async def upload_document(file: UploadFile = File(...), db: Session = Depends(get_db)):
    body = await file.read()   # 크기 계산용으로만 읽는다
    doc = Document(filename=file.filename or "unnamed",
                   content_type=file.content_type or "application/octet-stream",
                   size_bytes=len(body))
    db.add(doc); db.commit(); db.refresh(doc)
    return doc

1주차의 /·/health는 그대로 두고, 문서 메타 업로드·목록·단건 조회를 추가한다(다음 슬라이드). 파일 내용이 아니라 메타데이터만 저장한다.

1부 · 4단계 · 조회 엔드포인트

목록 · 단건 · 응답 스키마

main.py — 조회 & 스키마
class DocumentOut(BaseModel):
    id: int; filename: str
    content_type: str; size_bytes: int
    created_at: datetime
    class Config:
        from_attributes = True  # ORM→응답 변환

@app.get("/documents", response_model=list[DocumentOut])
def list_documents(db: Session = Depends(get_db)):
    return list(db.scalars(
        select(Document).order_by(Document.id.desc())))

@app.get("/documents/{doc_id}", response_model=DocumentOut)
def get_document(doc_id: int, db: Session = Depends(get_db)):
    doc = db.get(Document, doc_id)
    if doc is None:
        raise HTTPException(404, "document not found")
    return doc
세 엔드포인트
  • PPOST /documents — 업로드 → 메타 저장 → 201
  • GGET /documents — 목록(최신순, 배열)
  • GGET /documents/{id} — 단건(객체), 없으면 404

from_attributes = True가 ORM 객체를 응답 스키마로 자동 변환한다. id·created_at은 DB가 채우므로 refresh 후 응답에 담긴다.

2부 · 6단계

docker-compose.yml — 복붙 완결본

docker-compose.yml — web + PostgreSQL 16
services:
  db:
    image: postgres:16
    environment:
      POSTGRES_USER: ${POSTGRES_USER}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
      POSTGRES_DB: ${POSTGRES_DB}
    volumes:
      - pgdata:/var/lib/postgresql/data   # 데이터 영속화
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${POSTGRES_USER} -d ${POSTGRES_DB}"]
      interval: 5s
      timeout: 3s
      retries: 10
    ports:
      - "5432:5432"          # 로컬 DB 툴 접속용(선택)

  web:
    build: .                    # 현재 폴더의 Dockerfile로 빌드
    environment:
      DATABASE_URL: ${DATABASE_URL}
    depends_on:
      db:
        condition: service_healthy   # db healthy 후 web 시작
    ports:
      - "8000:8000"

volumes:
  pgdata:                       # 명명된 볼륨 선언
2부 · 6단계 · 지시어 해설

각 항목이 하는 일

항목역할
image: postgres:16DB는 직접 빌드하지 않고 공식 이미지를 그대로 사용
build: .web은 현재 폴더의 Dockerfile로 docpilot 이미지를 빌드
volumes: pgdata:...DB 데이터를 명명 볼륨에 저장 → 컨테이너 지워도 유지
healthcheckpg_isready로 DB가 진짜 접속 가능한지 반복 확인
depends_on: service_healthydb가 healthy가 될 때까지 web 시작을 대기
ports: "8000:8000"호스트 8000 → web 컨테이너 8000 매핑
${...}.env의 값을 주입 (자격증명·접속 URL)

depends_on: condition: service_healthy + healthcheck가 이번 구성의 핵심이다. 이게 없으면 web이 DB보다 먼저 떠서 접속 에러가 난다(흔한 함정).

git add Dockerfile main.py db.py \
