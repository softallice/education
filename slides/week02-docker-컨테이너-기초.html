개념 ① / 격리 기술

컨테이너는 게스트 OS가 없으니 가볍고 빠르다. "동일한 실행 환경을 통째로 패키징"이 핵심 — 12-Factor의 Dev/prod parity를 실현한다.

컨테이너는 게스트 OS 계층이 통째로 빠진다. 그래서 이미지가 MB 단위로 작고, 기동이 초 단위로 빠르다.

개념 ② / 이미지 · 레이어 · 레지스트리

틀 · 인스턴스 · 스택 · 원격 저장소

이미지 (Image)

실행에 필요한 파일시스템 스냅샷 + 메타데이터. 컨테이너를 찍어내는 틀.

컨테이너 (Container)

이미지를 실행한 인스턴스. 이미지 하나로 여러 컨테이너를 띄울 수 있다.

레이어 (Layer)

Dockerfile 명령마다 쌓이는 읽기 전용 레이어 스택. 캐시되고 이미지 간 공유된다.

레지스트리 (Registry)

이미지를 저장·배포하는 원격 저장소. Docker Hub · GHCR · ECR/GCR 등.

레이어가 캐시·공유되니 빌드가 빠르고 저장이 효율적이다.

개념 ② / 레이어 캐시 규칙 (중요)

자주 안 바뀌는 것을 먼저

순서가 캐시를 좌우한다

Dockerfile은 자주 안 바뀌는 것 → 자주 바뀌는 것 순으로 써야 레이어 캐시가 잘 먹는다.

그래서 의존성 설치(requirements.txt)를 소스 복사보다 먼저 한다.

캐시가 잘 먹는 순서
# 안 바뀜 → 캐시 O
COPY requirements.txt .
RUN  pip install -r requirements.txt

# 자주 바뀜 → 뒤로
COPY main.py .

소스만 고쳤을 때 pip install 레이어는 재사용되어 재설치를 건너뛴다 → 빌드가 훨씬 빠르다.

사전 준비 / 시작점

시작 전 체크

지난주 산출물: docpilot 로컬 앱 (main.py, requirements.txt, .gitignore) + GitHub 저장소

  • ✓Docker Engine 또는 Docker Desktop

1부 · 2단계

Dockerfile — 복붙 바로 빌드되는 완결본

Dockerfile — docpilot week02
# 슬림 베이스: 데비안 slim + Python 3.12
FROM python:3.12-slim

# 파이썬 런타임 권장 환경변수
ENV PYTHONDONTWRITEBYTECODE=1 \
    PYTHONUNBUFFERED=1

WORKDIR /app

# 의존성 먼저 복사·설치 (레이어 캐시 최적화 핵심)
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 앱 소스 복사 (자주 바뀌는 것은 뒤로)
COPY main.py .

# 문서화용 포트 노출 선언
EXPOSE 8000

# 시작 명령 — 컨테이너 밖 접속하려면 반드시 0.0.0.0
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
1부 · 2단계 · 지시어 해설

각 지시어가 하는 일

지시어역할
FROM베이스 이미지 지정 — python:3.12-slim은 풀 이미지보다 수백 MB 작다
ENVPYTHONDONTWRITEBYTECODE=.pyc 안 만듦 · PYTHONUNBUFFERED=로그 즉시 출력
WORKDIR작업 디렉터리를 /app으로 (이후 명령의 기준 경로)
COPY빌드 컨텍스트의 파일을 이미지 안으로 복사
RUN빌드 시점에 명령 실행 (의존성 설치)
EXPOSE포트 노출을 문서화 (실제 매핑은 docker run -p)
CMD컨테이너 시작 시 실행할 기본 명령
왜 0.0.0.0인가?

1주차엔 127.0.0.1을 썼다. 하지만 컨테이너 안에서 127.0.0.1에 바인딩하면 컨테이너 내부에서만 접근된다. 호스트에서 접속하려면 모든 인터페이스(0.0.0.0)에 바인딩해야 한다.

-p 8000:8000 = 호스트 8000 → 컨테이너 8000 (호스트:컨테이너). 1주차와 결과는 똑같지만 이제 컨테이너 안에서 돈다 — "한 번 빌드, 어디서든 실행".

과제 심화 / 이미지 최적화

멀티스테이지로 이미지 줄이기

Dockerfile.slim — builder에서 설치, 최종엔 결과만
FROM python:3.12-slim AS builder
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir \
    --prefix=/install -r requirements.txt

FROM python:3.12-slim
ENV PYTHONDONTWRITEBYTECODE=1 PYTHONUNBUFFERED=1
WORKDIR /app
COPY --from=builder /install /usr/local
COPY main.py .
EXPOSE 8000
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
bash — 빌드 & 크기 비교
docker build -f Dockerfile.slim \
  -t docpilot:0.2.0-slim .
docker images | grep docpilot
curl http://127.0.0.1:8000/health

제출물에 두 이미지의 크기(MB)와 차이를 기록한다. 기능은 그대로, 크기만 준다.
