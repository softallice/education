개념 ① / 멀티모달이란

모달리티는 데이터의 형태

"모달리티(modality)"는 데이터의 형태다 — 텍스트, 이미지, 음성, 비디오. 멀티모달 모델/시스템은 여러 형태를 함께 다룬다. docpilot 관점에서는 사용자가 스크린샷·다이어그램이나 음성 메모로 질문을 던질 수 있게 된다.

태스크입력 → 출력예 모델
이미지 캡셔닝 (image-to-text)이미지 → 설명 문장Salesforce/blip-image-captioning-base
자동 음성 인식 (ASR)오디오 → 텍스트openai/whisper-small
비전 질의응답 (VQA) / 멀티모달 LLM이미지 + 질문 → 답GPT-4o, Gemini(비전)
텍스트 임베딩문장 → 숫자 벡터sentence-transformers/all-MiniLM-L6-v2

이번 주는 이미지 캡셔닝 또는 ASR(택1) + 텍스트 임베딩(필수)에 집중한다.

개념 ② / Hugging Face 생태계

"깃허브 for ML" — Hub와 두 갈래 추론

Hub

모델·데이터셋·데모(Spaces)를 공유하는 "깃허브 for ML". 모델마다 카드(설명)와 태스크 태그가 있다.

transformers

모델을 내 컴퓨터/서버에서 직접 실행하는 파이썬 라이브러리. pipeline("task", model=...) 한 줄로 추론.

Inference API

HF가 대신 호스팅해 주는 HTTP API. 모델을 내려받지 않고 호출(Inference Providers).

임베딩 모델

sentence-transformers 계열이 대표적. 문장을 고정 길이 벡터로 바꾼다.

이번 주 흐름: transformers로 로컬 실행(1·2부) → Inference API로 매니지드 대안(3부)을 비교.

개념 ③ / 임베딩과 유사도 — RAG의 씨앗

뜻이 비슷하면 벡터도 가깝다

임베딩(embedding)

문장을 의미를 담은 숫자 벡터(예: 384차원)로 변환. 뜻이 비슷한 문장은 벡터도 가깝다.

코사인 유사도(cosine similarity)

두 벡터가 이루는 각도로 유사도를 잰다. 1에 가까울수록 비슷, 0이면 무관.

"강아지가 뛴다" → [0.12, -0.4, ...] "개가 달린다" → [0.10, -0.38, ...]

↑ 두 벡터의 코사인 유사도 높음(비슷)

"세금 신고 방법" → [-0.5, 0.2, ...]

↓ 코사인 유사도 낮음(무관)

week08 RAG에서는 "질문 임베딩"과 "문서 조각 임베딩"의 유사도로 관련 조각을 검색한다.

개념 ④ / 자체호스팅 vs 매니지드 트레이드오프

직접 돌릴까, 맡길까

기준로컬 transformers (자체호스팅)Inference API (매니지드)
초기 지연모델 다운로드·로딩 필요(수 초~분)없음(HTTP 호출)
비용하드웨어(특히 GPU)·전기호출량 기반 요금(무료 티어 있음)
데이터 프라이버시데이터가 서버 밖으로 안 나감외부로 전송됨
오프라인가능불가(네트워크 필요)
확장성직접 스케일링(week12 K8s)제공자가 처리

실무 감각: 개인정보·규제 데이터·대량 처리·오프라인이면 자체호스팅, 빠른 시작·간헐적 사용이면 매니지드.

1부 · 3단계

app/embeddings.py — 벡터 + 유사도

app/embeddings.py
from functools import lru_cache
import numpy as np
from sentence_transformers import SentenceTransformer
from app.config import get_settings


@lru_cache
def get_model() -> SentenceTransformer:
    # 모델을 최초 1회만 로딩(프로세스 내 재사용)
    s = get_settings()
    return SentenceTransformer(s.embedding_model)


def embed(texts: list[str]) -> np.ndarray:
    model = get_model()
    return model.encode(
        texts,
        normalize_embeddings=True,  # L2 정규화 → 내적=코사인
        convert_to_numpy=True)


def cosine_similarity(a, b) -> float:
    # 정규화된 두 벡터의 코사인 유사도(=내적)
    return float(np.dot(a, b))
핵심 설계

모델은 한 번만 로딩해 재사용한다(@lru_cache). 매 호출 로딩은 느리고 낭비.

normalize_embeddings=True로 L2 정규화하면 내적이 곧 코사인 유사도가 되어 계산이 간단해진다.

from pydantic import BaseModel, Field
from app import embeddings


class EmbedRequest(BaseModel):
    texts: list[str] = Field(
        ..., min_length=1, max_length=64)


class EmbedResponse(BaseModel):
    model: str
    dim: int
    vectors: list[list[float]]


@app.post("/embed", response_model=EmbedResponse)
async def create_embeddings(req: EmbedRequest):
    from app.config import get_settings
    vecs = embeddings.embed(req.texts)
    return EmbedResponse(
        model=get_settings().embedding_model,
        dim=int(vecs.shape[1]),
        vectors=vecs.tolist())

/caption — 이미지를 문장으로 (BLIP)

app/vision.py
import io
from functools import lru_cache
from PIL import Image
from transformers import pipeline
from app.config import get_settings


@lru_cache
def get_captioner():
    s = get_settings()
    return pipeline("image-to-text",
                    model=s.caption_model)


def caption_image(image_bytes: bytes) -> str:
    img = Image.open(io.BytesIO(image_bytes)).convert("RGB")
    result = get_captioner()(img)
    # [{"generated_text": "a dog running ..."}]
    return result[0]["generated_text"].strip()
app/main.py — 업로드 엔드포인트
from fastapi import UploadFile, File, HTTPException
from app import vision

ALLOWED = {"image/jpeg", "image/png", "image/webp"}
MAX_BYTES = 8 * 1024 * 1024  # 8MB


@app.post("/caption")
async def caption(file: UploadFile = File(...)):
    if file.content_type not in ALLOWED:
        raise HTTPException(415, "unsupported type")
    data = await file.read()
    if len(data) > MAX_BYTES:
        raise HTTPException(413, "file too large")
    return {"filename": file.filename,
            "caption": vision.caption_image(data)}

/transcribe — 음성을 받아쓰기 (Whisper)

app/audio.py
import tempfile
from functools import lru_cache
from transformers import pipeline
from app.config import get_settings


@lru_cache
def get_transcriber():
    s = get_settings()
    return pipeline(
        "automatic-speech-recognition",
        model=s.asr_model)


def transcribe(audio_bytes, suffix=".wav") -> str:
    with tempfile.NamedTemporaryFile(
            suffix=suffix, delete=True) as tmp:
        tmp.write(audio_bytes); tmp.flush()
        # Whisper 기본 30초 창 → 긴 입력은 청킹 필요
        result = get_transcriber()(
            tmp.name, chunk_length_s=30,
            return_timestamps=True)
    return result["text"].strip()

매니지드 — HF Inference API

app/hf_managed.py
from huggingface_hub import AsyncInferenceClient
from app.config import get_settings

# 레거시 api-inference URL을 직접 치지 말 것.
# InferenceClient가 최신 라우팅을 대신 처리한다.


def _client() -> AsyncInferenceClient:
    s = get_settings()
    if not s.hf_token:
        raise RuntimeError("HF_TOKEN is required")
    return AsyncInferenceClient(token=s.hf_token)


async def embed_via_api(texts) -> list:
    s = get_settings()
    client = _client()
    vectors = []
    for text in texts:
        vec = await client.feature_extraction(
            text, model=s.embedding_model)
        vectors.append(vec.tolist())
    return vectors
왜 AsyncInferenceClient인가

예전 serverless 엔드포인트는 Inference Providers(router.huggingface.co)로 이전됐고, all-MiniLM 같은 소형 모델은 무료 serverless로 안 뜨는 경우가 많다. huggingface_hub의 클라이언트가 최신 라우팅을 대신 처리한다.

이미지 캡셔닝은 client.image_to_text(image_bytes, model=...)로 동일하게 매니지드 호출할 수 있다.
