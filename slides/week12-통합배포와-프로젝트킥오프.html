개념 ② / 관측성 + 12요소(12-Factor) 재점검

배포는 끝이 아니라 시작이다

관측성(Observability)

로그·메트릭이 없으면 장애를 못 본다. 이번 주는 (1) 앱이 구조화(JSON) 로그를 stdout에 찍고, (2) kubectl logs로 수집하며, (3) /metrics로 기본 지표를 노출하는 데까지 한다. 무거운 스택(Prometheus/Grafana)은 프로젝트의 선택 과제로.

docpilot이 클라우드 네이티브 원칙(12-Factor)을 지키는지 표로 되짚는다. 설정·비밀은 코드가 아니라 환경에서 온다.

개념 ③ / 비용 · 성능 · 보안

운영형 배포의 세 가지 관점

비용

LLM 토큰이 가장 큰 변수. 모델을 gpt-4o-mini로 고정, 캐싱/상한(MAX_TOOL_TURNS)으로 제어. 무료 대안(HF Inference, 로컬 Ollama)도 존재.

성능

resources requests/limits로 스케줄링 안정화, HPA로 부하 대응. 파드가 죽어도 자동 복구되게.

보안

API 키는 절대 이미지·코드·git에 넣지 않는다. K8s Secret으로만 주입. 이미지에는 비밀이 없어야 한다.

핵심: 키는 없으면 기동 시점에 즉시 실패(fail-fast)해야 한다. 비밀을 코드에 하드코딩하지 않고 K8s Secret/환경변수로 주입한다.

1부 · 1단계 · 의존성·설정 정리 (12요소 III)

app/config.py — 새로 만들지 말고 재사용

⚠️ 여기서 app/config.py를 새로 만들지 말 것

평범한 os.environ 버전으로 덮어쓰면 week06~08에서 쌓은 필드가 전부 사라져 app.llm·app.rag 등의 import가 붕괴한다. 설정 모듈은 이미 week06에서 pydantic-settings 기반 Settings + get_settings()로 만들었다. 기존 파일을 재사용하고 두 가지만 점검한다.

  • 1openai_api_key가 필수(기본값 없음)로 선언되어, 키가 없으면 기동 시점에 즉시 실패(fail-fast)하는가.
  • 2database_url 기본값이 컨테이너 네트워크 기준(...@postgres:5432/...)인가. 아니면 매니페스트에서 DATABASE_URL을 주입.
bash — 설정 로딩 확인
python -c "from app.config import get_settings; \
get_settings(); print('config OK')"
확인

(필수 키가 있으면) config OK 가 출력된다. 키가 없으면 즉시 실패해야 정상 — 이 fail-fast가 곧 보안 규칙(비밀은 환경에서만)이다.

1부 · 3단계 · 프로덕션용 Dockerfile

비루트 실행 + graceful shutdown + COPY app ./app

Dockerfile
FROM python:3.12-slim

# 버퍼링 끄기 → 로그 즉시 stdout(12-Factor XI)
ENV PYTHONUNBUFFERED=1 \
    PYTHONDONTWRITEBYTECODE=1

WORKDIR /app

# 의존성 레이어 캐시: requirements 먼저
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 앱 소스 복사
COPY app ./app

# 비루트 유저로 실행(보안)
RUN useradd -m appuser
USER appuser

EXPOSE 8000

# SIGTERM에 정상 종료(12-Factor IX)
CMD ["uvicorn", "app.main:app", \
     "--host", "0.0.0.0", "--port", "8000"]

2부 · 5단계 · Secret으로 API 키 주입 (보안 핵심)

키를 YAML·git에 적지 말고 명령으로 생성

bash — Secret 생성 (파일·git에 남기지 않음)
kubectl create secret generic docpilot-secrets \

bash — 확인 (앞 7자만 복원)
kubectl get secret docpilot-secrets -n docpilot \

sk-... 처럼 키 앞부분이 복원되면 정상. Secret은 base64 인코딩일 뿐 암호화가 아니다 — 운영에선 Sealed Secrets/외부 KMS를 쓴다. --from-literal은 키를 파일로 남기지 않아 git 유출을 막는다.

2부 · 6단계 · PostgreSQL (pgvector 포함)

StatefulSet + pgvector/pgvector:pg16

k8s/02-postgres.yaml — Service + StatefulSet
apiVersion: v1
kind: Service
metadata: {name: postgres, namespace: docpilot}
spec:
  selector: {app: postgres}
  ports: [{port: 5432, targetPort: 5432}]
---
apiVersion: apps/v1
kind: StatefulSet
metadata: {name: postgres, namespace: docpilot}
spec:
  serviceName: postgres
  replicas: 1
  selector: {matchLabels: {app: postgres}}
  template:
    metadata: {labels: {app: postgres}}
    spec:
      containers:
        - name: postgres
          image: pgvector/pgvector:pg16
          env:
            - {name: POSTGRES_USER, value: docpilot}
            - {name: POSTGRES_PASSWORD, value: docpilot}
            - {name: POSTGRES_DB, value: docpilot}
          readinessProbe:
            exec: {command: ["pg_isready","-U","docpilot"]}
            initialDelaySeconds: 5
          volumeMounts:
            - {name: pgdata, mountPath: /var/lib/postgresql/data}
  volumeClaimTemplates:
    - metadata: {name: pgdata}
      spec:
        accessModes: ["ReadWriteOnce"]
        resources: {requests: {storage: 1Gi}}

kubectl apply -f k8s/02-postgres.yaml
kubectl rollout status \
  statefulset/postgres -n docpilot \

2부 · 7단계 · web Deployment (probe + resources + env)

ConfigMap→설정, Secret→키를 env로 주입

k8s/03-web.yaml — Deployment
apiVersion: apps/v1
kind: Deployment
metadata: {name: docpilot-web, namespace: docpilot}
spec:
  replicas: 2
  selector: {matchLabels: {app: docpilot-web}}
  template:
    metadata: {labels: {app: docpilot-web}}
    spec:
      containers:
        - name: web
          image: docpilot:1.0
          imagePullPolicy: IfNotPresent   # 로컬 이미지
          ports: [{containerPort: 8000}]
          envFrom:
            - configMapRef: {name: docpilot-config}
          env:
            - name: OPENAI_API_KEY          # 비밀만
              valueFrom:
                secretKeyRef:
                  name: docpilot-secrets
                  key: OPENAI_API_KEY
          resources:
            requests: {cpu: "100m", memory: "256Mi"}
            limits: {cpu: "500m", memory: "512Mi"}
          readinessProbe:
            httpGet: {path: /health, port: 8000}
            initialDelaySeconds: 5
          livenessProbe:
            httpGet: {path: /health, port: 8000}
            initialDelaySeconds: 15

3부 · 9단계 · 구조화 로그 + /metrics (관측성)

요청을 JSON 한 줄로, 지표를 /metrics로

app/main.py 에 추가 — 로그 미들웨어
import json, logging, sys, time
from fastapi import Request
from fastapi.responses import PlainTextResponse

logging.basicConfig(stream=sys.stdout, level="INFO",
                    format="%(message)s")
logger = logging.getLogger("docpilot")
_metrics = {"requests_total": 0, "errors_total": 0}


@app.middleware("http")
async def access_log(request: Request, call_next):
    """요청을 JSON 한 줄 로그로 남기고 카운터 증가."""
    start = time.time()
    _metrics["requests_total"] += 1
    try:
        response = await call_next(request)
    except Exception:
        _metrics["errors_total"] += 1
        raise
    dur_ms = round((time.time() - start) * 1000, 1)
    logger.info(json.dumps({
        "path": request.url.path,
        "method": request.method,
        "status": response.status_code,
        "duration_ms": dur_ms}))
    return response
app/main.py — /metrics
@app.get("/metrics", response_class=PlainTextResponse)
def metrics() -> str:
    """Prometheus 텍스트 포맷 기본 지표."""
    return (
        f"docpilot_requests_total {_metrics['requests_total']}\n"
        f"docpilot_errors_total {_metrics['errors_total']}\n")
