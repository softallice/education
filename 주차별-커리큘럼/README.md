## docpilot 러닝 프로젝트 성장표

같은 프로젝트가 매주 한 단계씩 자란다. "이번 주에 무엇을 추가하는가 → 무엇이 완성되는가"를 한눈에 정리했다.

| 주 | 추가되는 것 | 결과물 |
|----|------------|--------|
| 01 | FastAPI `GET /`("Hello, DocPilot") + `/health`, 로컬 uvicorn 실행 | 로컬 웹앱 |
| 02 | Dockerfile로 컨테이너화, 이미지 빌드·실행·레지스트리 푸시 | 컨테이너 이미지 |
| 03 | docker-compose: web + PostgreSQL, `/documents`(업로드 메타 DB 저장) | 멀티 컨테이너 |
| 04 | kind/minikube에 Deployment + Service 배포, kubectl 확인 | K8s 배포 |
| 05 | ConfigMap/Secret, probe, HPA, Ingress, 롤아웃/롤백 | 운영형 배포 |
| 06 | `/chat`: OpenAI/Gemini LLM 호출 + 스트리밍 + 프롬프트 | LLM 챗 |
| 07 | 멀티모달(이미지/음성) + Hugging Face 임베딩 모델 도입 | 멀티모달 기능 |
| 08 | RAG: 문서 임베딩 → 벡터DB(Chroma 또는 pgvector) → 검색 → `/ask` | RAG 서비스 |
| 09 | Agent 기초: tool/function calling, ReAct, `/agent`가 도구 호출 | 단일 Agent |
| 10 | 실제 API·DB·파일 도구 연동 + MCP 서버로 docpilot 도구 노출 | Agentic 서비스 |
| 11 | Multi-Agent: researcher + writer 협업으로 보고서 생성 | Multi-Agent |
| 12 | 전체를 K8s에 통합 배포 + 관측성 + 몰입형 프로젝트 킥오프 | 통합 배포 |

> 12주가 끝나면 docpilot은 **AI + Agent + 클라우드 네이티브**가 하나로 통합된 서비스가 되어 있다. 이후 4주 몰입형 프로젝트에서 이를 포크·확장한다([확장 경로](../03-평가와-몰입형-프로젝트.md#56-docpilot-확장-경로-권장) 참조).

## 주차 문서 공통 템플릿 (학생 안내)

모든 주차 문서는 아래 구조를 따른다. 매주 같은 흐름이므로, **"개념은 짧게, 실습은 따라 하기"**를 기대하면 된다.

- **Week N — 제목**: 상단에 "이번 주 한 줄"과 "docpilot 진화(이번 주 추가되는 것)"를 명시.
- **학습 목표**: 이번 주 끝에 할 수 있어야 하는 것들(체크리스트).
- **사전 준비**: 지난주 산출물, 필요한 도구·계정.
- **개념 (요약)**: 실습에 필요한 만큼만 짧게.
- **실습: 단계별 따라하기**: 명령·코드마다 "무엇을/왜" 한 줄 설명 + 실행 후 **확인**(기대 출력). 4교시 흐름에 맞춰 1부/2부/3부로 나뉠 수 있다.
- **트러블슈팅**: 증상 / 원인 / 해결 표.
- **이번 주 과제**: 제출물 명시(GitHub PR + 데모, [제출 방식](../03-평가와-몰입형-프로젝트.md#3-주차별-과제-제출-방식) 참조).
- **체크리스트 / 다음 주 예고**.

### 학생이 매주 기대할 것

- 강의 시간에 문서를 **위에서 아래로 따라 하면** 그 주 docpilot 기능이 완성된다.
- 각 명령에는 **왜 하는지**와 **성공했는지 확인하는 법(기대 출력)**이 붙어 있다.
- 막히면 트러블슈팅 표를 먼저 확인한다.
- 그 주 결과물을 브랜치 → PR로 제출하고 짧게 데모한다.
